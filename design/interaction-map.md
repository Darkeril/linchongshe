## 全局约定

### 导航结构

- **底部 TabBar（定义在 `screens-part1.jsx` 的 `AppTabBar`）**：5 项
  - `home`（首页）
  - `market`（市集）
  - `publish`（中间大按钮，跳发布流程）
  - `msg`（消息，含红点 badge）
  - `me`（我）
- TabBar 是液态玻璃胶囊，**浮在内容之上**（`position: absolute`），首页 / 市集 / 消息 / 个人主页 4 个 Tab 都包含该组件；详情类页面（NoteDetail / ProductDetail / Chat 等）**不出现 TabBar**。
- **顶层菜单键（汉堡 Icon）**：首页、市集、消息、个人主页左上都有三横线图标 → 触发 `DrawerScreen`（侧边抽屉）。

### 全局手势 / 交互

- **返回**：除 Tab 页外，所有页面左上都是返回箭头（TopBar `left="back"`）；AddPet 是 `left="close"`（关闭）。
- **关闭 / 取消**：发布页顶部是文字"取消"（非箭头）。
- **下拉刷新**：设计稿未画，**需要与产品确认刷新范围**（首页瀑布流 / 消息列表 / 订单列表是否统一）。
- **上拉加载**：瀑布流底部有"到底啦"静态文案，**是否存在分页加载、分页参数、骨架状态设计稿无**。
- **左滑手势**：订单列表 / 消息列表 / 地址列表是否支持左滑出"删除/置顶"，设计稿无。
- **双击点赞**：小红书式双击点赞，设计稿无标注。

### 全局状态

- **登录态**：登录页存在，但首页 / 商品详情 / 话题页是否允许游客访问？未登录点关注 / 点赞 / 评论 / 购买的降级路径设计稿无。
- **主题**：设计稿 3 套色板（cream / peach / latte），但 Settings 里有"深色模式开关"和 Drawer 里"夜间模式 · 跟随系统"——**这是色板切换，还是真正的深色模式？两套标识？**
- **网络态**：`EmptyStatesScreen` 有一个「网络偷偷跑开啦」的空态，但具体触发时机、全局 toast 还是整屏、重试行为，设计稿无。
- **时间/地区**：截图里有"上海 · 徐汇"、"北京·朝阳"、"杭州·西湖"等；定位权限、跨城体验未定义。
- **货币 / 语言**：Settings 语言行只写"简体中文"，是否支持多语言？货币是否固定 CNY（钱包写了 CNY，但市集未明示）。

## 页面：Splash 开屏页（screens-part1.jsx）

### 布局概览
全屏渐变（surface → bg → primarySoft），居中 Logo + 品牌名 + Slogan，底部进度条 + "LOADING · v2.0"。

### 入口
- App 冷启动。

### 出口
- 加载完成 → Login 或 Home（取决于登录态 / 是否首次启动）
- 首次启动 → Onboarding

### 主要控件 / 交互
- 装饰爪印（纯视觉）
- 核心徽标（圆形渐变）
- 底部加载条（CSS 动画，时长 1.8s）

### 状态
- 只有加载态，**无超时 / 失败态**。

### ❓ 业务盲点
- 冷启动停留多久？是固定时长、接口就绪跳转、还是两者取其大？
- 若首屏接口（如配置、强更、登录校验）失败，是否卡在 Splash？兜底文案 / 重试按钮在哪？
- 首次启动 vs 老用户启动，跳转路径差异？（首次 → Onboarding → Login；老用户 → Home）设计稿无判断逻辑。
- 强制升级、维护公告、合规弹窗（隐私政策二次确认）是否在这里拦截？
- 启动图是否需要按色板联动？3 套色板在此如何呈现？

## 页面：Onboarding 新手引导（screens-part10.jsx）

### 布局概览
4 页轮播引导，每页一个 emoji 主视觉 + 标题 + 描述 + 指示点 + CTA 按钮。

### 入口
- 首次冷启动（Splash 之后）

### 出口
- 点"下一步"→ 下一页
- 第 4 页点"开始我的云养宠之旅 🐾" → Login 或 Home
- 顶部"跳过 ›" → 直接跳出

### 主要控件 / 交互
- 4 张幻灯片（emoji 轮播）
- 圆点指示器（动画过渡）
- CTA 按钮（根据 step 变文案）
- 右上角"跳过 ›"

### 状态
- 无加载态 / 无错误态（纯本地 UI）

### ❓ 业务盲点
- Onboarding 只展示一次还是每次大版本更新都弹？用什么 key 存储？
- "跳过"和"完成"是否都走同一条跳转路径？
- 每页是否带埋点（曝光 + 点击）用于后续转化优化？
- 引导页结束后，若用户未登录，直接进 Home 还是必须登录？

## 页面：Login 登录选择页（screens-part1.jsx）

### 布局概览
顶部背景块 + 浮动宠物头像群，中部 Logo + 欢迎文案，底部两个按钮（微信一键登录 / 手机号登录注册），最底部协议勾选。

### 入口
- Splash / Onboarding 结束
- 从任何需要登录的入口（评论 / 点赞 / 关注 / 购买）跳转进来

### 出口
- 微信登录成功 → Home 或 BindPhone（补绑手机）
- 手机号登录 → 手机号/验证码输入页（设计稿缺）
- 点"用户协议" / "隐私政策" → WebView

### 主要控件 / 交互
- 微信按钮（主 CTA，渐变）
- 手机号按钮（次 CTA，描边）
- 协议勾选 Checkbox（受控状态）
- 两处协议链接（文字高亮）

### 状态
- 协议未勾选时是否禁用按钮？设计稿无禁用态；实际小红书 / 抖音流派是"点按钮→弹 bottomsheet 二次确认勾选"。

### ❓ 业务盲点
- **没有手机号输入页和短信验证页的设计稿**——必补。
- 微信一键登录在 iOS（非微信环境）如何降级？
- 若微信未授权手机号，是否走微信 openid → 补绑手机号页面？
- 登录失败的错误文案、频率限制文案、封号提示文案？
- 找回密码 / 注册新账号 / 国际手机号 是否支持？
- 一个微信能否关联多个爱宠号？同一手机号能否注册多次？
- 登录后是否强制完善资料（头像、昵称）？是否需要先绑宠物？
- 第三方登录（Apple ID、抖音 OAuth）未在设计稿出现，未来是否要求合规必须有 Apple 登录？

## 页面：Home 首页瀑布流（screens-part2.jsx）

### 布局概览
顶部：汉堡菜单 + 三 Tab（关注 / 发现 / 附近）+ 搜索 Icon；下方一排话题胶囊（🔥热榜 / 🐕狗狗 / 🐈猫咪 …）；主体是双列瀑布流卡片；底部 TabBar。

### 入口
- 登录成功
- TabBar 点"首页"
- 应用任意位置深链回首页

### 出口
- 点话题胶囊 → Topic（话题聚合页）
- 点搜索 Icon → Search
- 点卡片 → NoteDetail
- 点汉堡 → Drawer
- 点卡片头像 / 作者 → 对方 Profile
- 双击 / 点心形 → 点赞（本地乐观更新，颜色变 warn）
- Tab 切换：关注 / 发现 / 附近 → 对应列表
- TabBar → Market / Publish / Msg / Me

### 主要控件 / 交互
- 汉堡按钮（打开抽屉）
- Tab 文字（激活态字大 + 底部横线，非激活态灰）
- 搜索入口（圆形 Icon）
- 话题胶囊（首个默认选中、primary 色；可横滑）
- 双列瀑布流卡片：封面图、话题角标（可选）、标题（2 行省略）、作者头像+昵称、心形+数字（k 转换 `2341 → 2.3k`）
- "到底了，歇会儿吧 🐾"兜底文案

### 状态
- 心形点击状态（受控，从 mock 数据里读初始值）
- 没有"加载中 / 列表为空 / 网络失败"的分别设计

### ❓ 业务盲点
- **关注 Tab 为空时怎么展示？**（新用户还没关注任何人）
- 关注 / 发现 / 附近三 Tab 的数据源、排序规则、分页策略分别是什么？
- "附近"需要定位权限，首次进入是否拉起系统权限弹窗？定位失败的兜底？
- 话题胶囊的数据是写死（热门 6 条）还是运营后台可配？"🔥 热榜"含义？
- 点赞是否需要登录？未登录点赞怎么引导？
- 瀑布流上拉加载到底 / 分页大小 / 预加载图片的策略？
- "不感兴趣 / 举报笔记"等长按菜单是否要提供？
- 内容风控：涉政、色情、虐宠图如何拦截？首页曝光是否走审核过的池？
- 图片加载失败的占位（设计稿有 PetPlaceholder，但真图失败呢？）
- 埋点：卡片曝光 / 点击 / 停留时长是否全覆盖？

## 页面：Search 搜索发现页（screens-part6.jsx）

### 布局概览
顶部搜索框（带返回箭头 + 输入框 + "搜索"按钮），主体含最近搜索 chips、🔥 爱宠热榜（前 6）、🌈 逛逛分类（6 格）、💡 你可能想了解。

### 入口
- Home 右上角搜索 Icon
- Market 顶部搜索框（可能共用也可能独立）
- Profile / FollowList / Messages 头部搜索按钮

### 出口
- 输入关键字 → 搜索结果页（**设计稿缺**）
- 点热搜词、历史词、分类卡、猜你想搜标签 → 搜索结果
- 点垃圾桶 Icon → 清空历史（是否二次确认？）
- 左上箭头 → 返回

### 主要控件 / 交互
- 返回箭头
- 搜索输入框（带 🐾 占位尾标、primary 色放大镜）
- "搜索"文字按钮
- 历史 chips（灰底）
- 热榜列表（前 3 位 warn 色编号，HOT 角标）
- 分类卡（6 格 2×3 网格，渐变柔和）
- 猜你想搜（primarySoft 底）

### 状态
- 默认态（非输入态、有历史）
- 输入中（有自动联想？设计稿无）
- 搜索结果页不存在

### ❓ 业务盲点
- **搜索结果页缺失**——必补。需要定义：是否分 Tab（笔记 / 用户 / 商品 / 话题）？默认排序？筛选项？
- 搜索联想（suggest）接口是否存在？分词策略？
- 历史搜索：本地存 vs 服务端存？多设备同步？条数上限？删除某一条交互是长按还是编辑态？
- 热搜榜单更新频率"每日 9:00"——是写死还是后台配？时区如何？
- 分类卡跳转的目标（EmptyStatesScreen 提到的"养犬/养猫"）是 Topic 还是专门的分类页？
- 搜索需要登录吗？未登录的热榜限流？
- 搜索敏感词过滤：违禁、辱骂、政治敏感如何拦截？

## 页面：NoteDetail 笔记详情页（screens-part2.jsx）

### 布局概览
顶部悬浮导航：返回、作者胶囊（头像+昵称+关注按钮）、更多（三点）；主体滚动区：轮播大图（带页码 1/4 + 指示点）、标题、正文、话题 chips、编辑时间；分隔条；评论区（前 3 条）；底部操作栏：评论占位输入 + 心/藏/评计数。

### 出口
- 返回 → 上一页
- 作者头像/昵称 → 作者 Profile
- "+ 关注" → 关注态切换（需登录）
- 三点 → 底部动作表（分享 / 举报 / 屏蔽 / 不感兴趣 / 复制链接）**设计稿缺该面板**
- 话题 chips → Topic
- 评论区 "说点什么吧..." → 唤起评论输入（全屏？浮层？设计稿无）
- 心形 → 点赞
- 书签 → 收藏（可能进收藏夹选择，但 NoteDetail 没有选文件夹交互）
- 评论 Icon → Comments（完整评论页）
- 滑动到底 → 推荐相关笔记？设计稿无

### 主要控件 / 交互
- TopBar（透明，毛玻璃胶囊）
- 图片轮播（指示器：圆点 + 胶囊）
- 话题 chip（primarySoft 底色）
- 评论卡（头像+名+文字+点赞）
- 底部三个 Icon（心、藏、评）带受控数字

### 状态
- `liked` / `saved` 本地 state
- 加载态：图片未加载？正文加载？评论加载？**全都缺**
- 笔记被删除 / 作者被封 / 仅粉丝可见的兜底？缺

### ❓ 业务盲点
- 笔记支持的媒体类型：图文、视频、图文混排、长图？
- 视频是否自动播放？静音？流量提示？
- 图片查看：点击是否进入大图浏览器（捏合、保存）？
- 收藏按钮点击是否立即入默认收藏夹，还是弹出收藏夹选择？（Favorites 页明确有多收藏夹）
- 评论数是 128，底部输入框字数上限？表情、@、话题、图片评论是否支持？
- 举报/屏蔽/拉黑的触发入口、反馈文案、后端流程？
- 点赞数字是否实时刷新（心跳？轮询？）？
- 作者"+ 关注"按钮在已关注态下是什么表现（隐藏 / 变成"已关注"）？
- "编辑于 2 小时前 · 来自上海" — 位置信息用户是否可隐藏？
- 数据同步：点完赞 → 返回 Home 时列表里的数字要不要跟着变？

# 全局盲点清单（跨页面业务规则）

以下是贯穿多个页面、设计稿无法独立解释、但后端/前端/产品必须共同确认的问题。目标≥20 条，实际列出 30 条。

1. **登录态层级**：哪些页面允许游客浏览？哪些动作强制登录（点赞、关注、评论、收藏、下单、私信、发布、签到）？统一的"未登录拦截"交互是 bottomsheet 还是跳 Login？
2. **未登录 → 动作 → 登录回调链路**：用户点"+ 关注" 跳 Login，登录完成后如何精准回跳并完成原动作？
3. **深链路由（Deep Link）**：分享笔记 / 商品 / 话题 / 直播间 / 领养卡 / 走失卡时，外部打开 App 的路径表？iOS Universal Link + 安卓 App Link + 微信小程序转跳？
4. **推送通知分类**：赞 / 评论 / 粉丝 / 私信 / 订单 / 物流 / 签到提醒 / 疫苗到期 / 领养 / 走失寻宠 / 直播开播 / 活动 —— 分类开关设计稿无，iOS 通知 category 架构缺。
5. **风控 & 审核**：发布笔记、评论、私信、头像、昵称、话题、商品、直播、礼物、寻宠 —— 各有文本 / 图片 / 视频的机审 + 人审策略未定义。
6. **多端数据同步**：iOS / Android / 小程序同时登录时，消息已读、购物车、草稿箱、浏览历史是否实时同步？
7. **乐观更新回滚策略**：点赞 / 收藏 / 关注在客户端先改数字，接口失败时如何回滚并提示？CLAUDE.md 要求走乐观更新但未定义失败回滚 UI。
8. **视频 / 图片上传中台**：发布笔记、评论图、聊天图、头像、相册上传是否走同一套上传中台？进度 / 取消 / 重试？压缩策略？CDN 回源？
9. **货币体系整合**：现金（¥）/ 爪币（🐾）/ 爱宠豆 / 积分 / 成长值 / 优惠券 / 红包 —— 6 种"财产"混杂，兑换规则、账本、对账口径必须一致。
10. **内容举报 & 处置时效**：举报后多久响应？违规处置（隐藏、删除、禁言、封号）通知用户的渠道？
11. **拉黑/屏蔽双边可见性**：A 拉黑 B 后，B 是否知情？B 能否看到 A 的内容？评论、搜索、私信、粉丝列表分别如何隔离？
12. **青少年模式 / 未成年保护**：强制实名？时长限制？内容过滤？支付限制？直播礼物？
13. **隐私开关粒度**：在线状态、位置、粉丝/关注列表、收藏、赞过、相册、宠物档案、手机号 —— 每项独立开关？统一隐私中心？
14. **"我关注的话题 / 关注的宠物 / 关注的主播"三类关注的聚合**：分开的入口还是统一的关注中心？
15. **推荐与标签体系**：宠物品种 / 宠物年龄 / 用户性别 / 地区 / 话题 / 购买偏好 —— 推荐模型所需特征的采集和存储规则？
16. **社区 × 电商的内容流转**：笔记关联商品 → 商品详情的买家秀是否自动聚合这些笔记？带来 GMV 归因？
17. **多商家架构**：Cart 和 Orders 已按商家分单，但店铺主页、商家评分、客服直连、保证金、结算这些页面和接口设计稿零覆盖。
18. **直播与电商的法规合规**：直播带货需明确"小黄车"规范、价格一致性、退换货承诺。
19. **跨境宠物 / 异宠的法规限制**：猫狗有检疫、限养品种各城不同；小宠、水族、飞鸟的禁养清单。
20. **公益 × 盈利模块的边界**：Adopt（领养）、Lost（走失）是公益免费，Adopt 的送养审核与违法贩卖的边界？禁止收取任何费用的声明？
21. **会员与签到奖励的用户心智冲突**：Lv / VIP / 爪币 / 签到 —— 防止"氪金薅羊毛"矛盾。
22. **错误兜底体系**：超时 / 断网 / 接口 500 / 限流 / 鉴权失效 —— 各自的 UI 表现（全屏 / toast / snackbar / 顶部条）？重试和自动退出机制？
23. **首屏性能指标**：CLAUDE.md 要求骨架屏、虚拟滚动、CSS 动画 —— 每个瀑布流 / 列表都要统一骨架屏样式，设计稿未提供。
24. **iOS 审核合规点**：
    - Apple ID 登录必须提供
    - 微信登录前的"同意协议"弹层（不主动勾选）
    - IAP 支付规则（虚拟货币必须走 IAP；实物可走微信 / 支付宝）
    - 儿童模式、已删用户数据导出
25. **国际化 / 方言**：Settings 显示"简体中文"，未来是否支持繁体、英文、东南亚？货币、时区、日期格式？
26. **搜索历史和足迹的本地 vs 云端**：清空的范围？多端同步的延迟？
27. **消息数字合计的冒泡**：TabBar 消息 3 红点 vs Messages 内各分类 N+M+P 的合计算法？官方通知是否计入？
28. **邀请返利 / 裂变机制**：Drawer "邀请好友 各得 15 元券"——具体裂变链路、奖励到账、防刷？设计稿缺。
29. **离线与弱网体验**：无网时进入是否展示离线缓存（最近浏览）？评论能否先离线排队上传？
30. **埋点 & 分析体系**：每个页面曝光 / 控件点击 / 停留时长 / 来源 / 离开 —— 需要统一埋点命名表和 Funnel 设计。若缺失，所有运营优化无从谈起。
